Sources utilisées

Les anciens travaux réalisés

Un grand nombre de composants avaient déjà été réalisés dans d'autres travaux:

  • Le dialog,
  • La recherche,
  • La vérification de courriel,
  • La méthode toHtml,
  • La bannière.

Tout était déjà bien compris mais réutiliser des bouts de code aide vraiment à la compréhension. Il faut resélectionner les éléments html, se remémorer le code, supprimer tout ce qui est superflu ou qui était utile pour un ancien projet pour que tout fonctionne correctement. Ça m'a permis de réviser et pratiquer des composants divers.

MDN

Site web de MDN

MDN est une très bonne source d'information pour le HTML, le CSS, le JavaScript et l'accessibilité. Je m'en sers tous les jours, à chaque fois que j'ai besoin de plus de détails sur un éléments ou pour savoir comment l'utiliser.

Tailwind

Site web de Tailwind

Cette ressource est essentielle pour connaitre la façon d'écrire une class CSS dans le HTML grâce à Tailwind.

Ce que je comprends est que les règles de styles sont prédéfinies et qu'on les utilise en appelant la class dans le HTML. Les valeurs non définies par Tailwind peuvent être écrites entre crochets, les statuts et les media-queries sont définis avant la class avec deux points. Tout est combinable.

Pour chercher une règle de style en particulier, il suffit de rechercher sur le site web. La documentation est très complète.

On peut définir un thème (couleurs et typographies) dans tailwind.config.

Tailwind, c'est vraiment très pratique!!

Chat GPT

Je me sers parfois de Chat GPT pour corriger un bout de code ou me l'expliquer si je n'ai pas compris. Je ne trouve pas utile de tout générer par AI, c'est même un peu dommage et ça fonctionne rarement comme on le voudrait. Par contre, quand on supprime une parenthèse par erreur sur 80 lignes de code, on est bien content qu'il existe.

Les cours

Ce qui m'a le plus aidé, c'est les cours de manière générale. Ce que j'ai retenu de cette session, c'est principalement l'utilisation de différents "langages" pour éviter d'écrire du CSS et gagner du temps: Tailwind et Bootstrap.

L'utilisation du data-dialog pour faire fonctionner les modales est vraiment très utile. Ce que je comprends là dedans, c'est que le code peut être réutilisé simplement grâce à l'attribut data-dialog et un id correspondant.

Le cart est, je trouve, le plus gros morceau de la session, mais j'ai vraiment apprécié essayer de le faire fonctionner. Je comprends de ça que les classes JS peuvent s'imbriquer les unes dans les autres, qu'on peut afficher un produit de plusieurs manières et à plusieurs endroits en le définissant qu'une seule fois.
